# Working Memory Protocol

## Active Task Management

### Task Organization
- Current projects and priorities
- Task dependencies and relationships
- Progress tracking and milestones
- Resource allocation and constraints

### Collaboration Workflow

#### Task Handoffs
- Clear context and background information
- Current status and progress updates
- Next steps and action items
- Required resources and dependencies

#### Documentation Standards
- Task descriptions and requirements
- Decision logs and rationale
- Progress updates and blockers
- Completion criteria and acceptance tests

## Memory Management

### Context Preservation
- Task-specific knowledge and insights
- Technical decisions and trade-offs
- Lessons learned and best practices
- Stakeholder feedback and requirements

### Knowledge Transfer
- Project handoff documentation
- Technical knowledge base updates
- Process improvement recommendations
- Team learning and skill development

## Active Directory Structure

### `/active/[task-name]/`
Each active task gets its own directory containing:
- `context.md` - Task background and requirements
- `progress.md` - Current status and updates
- `decisions.md` - Technical and design decisions
- `resources/` - Related files and assets

### `/archive/`
Completed tasks are moved here for future reference:
- Organized by completion date
- Includes final status and outcomes
- Lessons learned documentation
- Reusable assets and templates
