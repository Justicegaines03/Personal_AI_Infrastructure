# Core Architectural Guidelines

## Principles

- **Modularity**: Build systems with clear, well-defined interfaces
- **Scalability**: Design for growth and changing requirements
- **Maintainability**: Write code that is easy to understand and modify
- **Security**: Security considerations at every layer
- **Performance**: Optimize for speed and efficiency
- **Reliability**: Build robust systems that handle failures gracefully

## Best Practices

- Follow established design patterns
- Document architectural decisions
- Use version control effectively
- Implement proper testing strategies
- Monitor and measure system performance

## KEEP IN MIND

- If you need to write .md files for instructions or an overview, then the solution is too complicated
